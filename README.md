# ✈️ AI Tour Guider

A full-stack AI-powered travel planning web application built with **React**, **FastAPI**, and **Groq LLaMA**. Plan personalized trips, track your visits, store memories, and get real-time emergency assistance — all in one place.

## 🚀 Features

### Trip Planning
- 🌍 **Smart destination search** with real-time autocomplete (OpenStreetMap Nominatim)
- 📍 **Multi-spot selection** with place photos (Unsplash API)
- 🗺️ **Route optimization** using Nearest Neighbor + 2-opt TSP hybrid algorithm
- 💰 **3 budget tiers** — Budget, Mid-range, Luxury
- 📅 **Flexible duration** with AI-suggested stay durations
- 🚌 **Transport options** — Bus, Rail, Air with cost & timing
- 📋 **Day-by-day itinerary** generated by Groq LLaMA

### During the Trip
- ✅ **To-do checklist** — tick places as you visit them
- 📷 **Photo upload** per spot → stored in Cloudinary → auto-saved to Memories
- 💬 **Comments** per spot → saved to database
- 💡 **AI suggestions** after each tick for what to do next
- 🚨 **Emergency panel** — hospitals, police, hotels, helpline numbers
- ❌ **Cancel trip** — enter home address → nearest bus/rail/air station

### History & Memories
- 🗂️ **Trip history** — all planned trips with visited/missed status per spot
- 📸 **Memories wall** — photo gallery from all trips + manual uploads
- 🔁 **RAG suggestions** — AI suggests missed spots from previous visits
- ⚠️ **Problem reporting** — log issues per trip, used to improve future suggestions

### Account
- 🔐 **Auth** — email/password via Supabase Auth
- 👤 **Profile** — name, city, phone, preferred language
- ⚙️ **Preferences** — travel style, interests (used for AI personalization)
- 🌐 **10 languages** — English, Hindi, Telugu, Tamil, Kannada, Malayalam, Bengali, Spanish, French, Arabic

### 3. Supabase database setup

Run this SQL in your Supabase SQL editor:

```sql
-- User profiles
create table profiles (
  id uuid references auth.users primary key,
  name text, phone text, city text,
  travel_style text, interests text[], languages text,
  created_at timestamptz default now()
);

-- Trips
create table trips (
  id uuid default gen_random_uuid() primary key,
  user_id uuid references auth.users,
  place text, optimal_route text[], budget text,
  days int, transport text, lang text default 'en',
  created_at timestamptz default now()
);

-- Spots per trip
create table trip_spots (
  id uuid default gen_random_uuid() primary key,
  trip_id uuid references trips,
  spot_name text, visited boolean default false,
  comment text, photo_url text, visited_at timestamptz
);

-- Memories
create table memories (
  id uuid default gen_random_uuid() primary key,
  user_id uuid references auth.users,
  trip_id uuid references trips,
  title text, description text, photo_url text,
  created_at timestamptz default now()
);

-- Past problems for AI learning
create table past_problems (
  id uuid default gen_random_uuid() primary key,
  user_id uuid references auth.users,
  trip_id uuid references trips,
  problem_text text, created_at timestamptz default now()
);

-- Enable Row Level Security
alter table profiles      enable row level security;
alter table trips         enable row level security;
alter table trip_spots    enable row level security;
alter table memories      enable row level security;
alter table past_problems enable row level security;

-- RLS Policies
create policy "own profile"   on profiles    for all using (auth.uid() = id);
create policy "own trips"     on trips       for all using (auth.uid() = user_id);
create policy "own spots"     on trip_spots  for all using (
  trip_id in (select id from trips where user_id = auth.uid())
);
create policy "own memories"  on memories    for all using (auth.uid() = user_id);
create policy "own problems"  on past_problems for all using (auth.uid() = user_id);
```

## 🧠 Route Optimization Algorithm

```
Phase 1 — Nearest Neighbor (greedy)
  Start at first spot → always go to closest unvisited spot
  Time complexity: O(n²)

Phase 2 — 2-opt improvement
  Try reversing every sub-segment
  Keep if shorter total distance
  Eliminates route crossings

Result: Near-optimal route in O(n²) time
Suitable for 2-10 spots (typical trip planning range)
```

- Supabase Row Level Security (RLS) on all tables
- Service key used only in backend (never exposed to browser)
- Anon key used in frontend (read-only public access)
- `.env` files excluded from version control
